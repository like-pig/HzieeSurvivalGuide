Java Scanner Notes Share

最近入了Java的坑 在使用Scanner类的时候发现了一点小问题 分享下解决经验 以下是原代码

                                    
import java.util.*;
 
public class Mathmatic {
                                             
public static void main(String[] args) {
    Mathmatic math = new Mathmatic();
    int result;
    result = math.compare();
    System.out.println(result);
    result = math.isPrime();
    System.out.println(result);
}
                                                 
public int compare() {
    int num1,num2;
    Scanner input = new Scanner(System.in);
    System.out.println("Please input the first number:");
    num1=input.nextInt();
    System.out.println("Please input the second number:");
    num2=input.nextInt();
        input.close();
    if(num1==num2) {
        return 0;
    }else if(num1> num2) {
        return 1;
    }else if(num1< num2) {
        return -1;
    }
        return 2;
}
                                                 
public int isPrime() {
    int prime;
    Scanner input = new Scanner(System.in);
    System.out.println("Please input a number:");
    prime=input.nextInt();
    for(int i=2;i< prime;i++) {
     if(prime%i==0) {
        return 0;
        }
    }
        return 1;
   }
}



以上代码在运行中 如果调用Compare函数之后再调用isPrime函数 就会出错 以下是输出内容
Exception in thread "Main" java.util.NoSuchElementException
at java.util.Scanner.throwFor(Unkown Source)
at java.util.Scanner,.next(Unknown Source)
at java.util.Scanner,.nextInt(Unknown Source)
at java.util.Scanner,.nextInt(Unknown Source)
at shuxue.Mathmatic.isPrime(Mathmatic.java:38)
at shuxue.Mathmatic.main(Mathmatic.java:12)

错误定位

                                
                num2=input.nextInt();
               -->input.close();<--

这是因为在某个函数中 声明了一个Scanner对象 并且调用了close 可能由于所有的Scanner类共享一个输入流 当其中一个调用关闭函数的时候所有的Scanner对象都关闭了

解决方法：程序末尾 使用input.close 就好了
